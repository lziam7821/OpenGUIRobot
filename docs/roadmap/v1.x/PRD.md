## 1. 版本概述

v1.0 让平台**生产可用**，v1.x 让平台**深度差异化**。这一组特性不必同步发布，每个独立成 minor：

- **v1.1**：业务地图插件（L6） — 给业务流程稳定的客户上"模板 + Plan + ReAct"
- **v1.2**：视频 → 操作图谱抽取 — 把用户操作录屏自动转成可执行图谱
- **v1.3**：IDE 插件（VSCode + JetBrains） — 在编辑器内原生写自然语言用例
- **v1.x.E**：商业版差异化能力（SSO 进阶、审计日志、企业 SLA、专属支持）

每个特性独立 PRD（本文件是路线图汇总，具体细化在小版本时再展开）。

## 3. v1.1 业务地图插件（L6）

### 范围 In

- 业务模板 schema（JSON / YAML 双支持）
- 模板编辑器（Dashboard 内 ProTable + ProForm + 自定义图谱编辑）
- 向量 + 关键词混合匹配
- Plan + ReAct 协同执行（命中模板 → 走 Plan，未命中 → 回到 Code-as-Action）
- 模板版本管理 + 灰度发布
- 模板贡献市场（社区）
- 模板与 v0.3 操作图谱的双向映射

### 关键 FR

1. **模板 schema 与变量化**：链路里挖参数槽（`{类型: "保姆", 经验: "3 年"}`）
2. **模板匹配器**：Orchestrator 之前注入；命中置信度 < 阈值时走兜底
3. **Plan 生成 zero-token**：模板展开即 Plan，不用 LLM
4. **ReAct 执行**：仅元素定位 / 异常处理调用 LLM
5. **模板版本灰度**：v1 / v2 共存，按 % 流量切

### 验收标准

- 在 5 家业务流程稳定的客户上验证：覆盖率 ≥ 80% 的 case 走模板路径
- 业务模板路径的 LLM 成本相比 Code-as-Action 路径下降 ≥ 60%
- 模板编辑器 7 天上手率 ≥ 70%

## 4. v1.2 视频 → 操作图谱抽取

### 范围 In

- 录屏文件（mp4 / mov）→ 帧抽取 → 操作识别（点击 / 滑动 / 输入）→ 图谱节点 + 路径
- 多模态 grounding 模型识别页面元素与动作
- 人工审核 UI（Dashboard 中"录屏导入"页）
- 与 v0.3 操作图谱无缝对接
- 与 v1.1 业务模板可一键提升为模板

### 关键 FR

1. **抽取 pipeline**：opencv 抽帧 + Qwen-VL grounding + 动作识别 + DOM diff（如有）
2. **置信度评分**：每个节点 / 边带 0–100 分
3. **review UI**：把抽取结果做成可点选 / 修正 / 合并的 ProTable
4. **批量导入**：支持目录级批量
5. **可逆性**：图谱节点保留来源录屏链接，可回看

### 验收标准

- 真实录屏 100 段（每段 30s–3min），节点级抽取准确率 ≥ 80%
- review 工时 ≤ 录屏时长的 30%（即 1 小时录屏需 ≤ 18 分钟 review）
- 抽取产物能直接被 explore 复用

## 5. v1.3 IDE 插件

### 范围 In

- VSCode 插件（TypeScript）
- JetBrains 插件（Kotlin / IntelliJ Platform，覆盖 IDEA / PyCharm / Android Studio）
- 功能：
  - 在编辑器内写 case YAML，按快捷键触发 explore
  - 实时预览设备屏幕（截图流）
  - 选中代码块 → 一键 Heal
  - 直接打开 evidence（截图 / DOM）
  - LSP-style 自动补全（locate("...") 提示历史候选）

### 关键 FR

1. **远端 API 通信**：通过 OGR HTTP API 与本机或集群通信
2. **设备实时屏幕**：30 fps 截图流（WebSocket）
3. **LLM in-place suggest**：在 case YAML 中按 alt+i 触发"补全意图"
4. **跳转**：从固化代码 → 对应 case YAML → 对应 explore 记录

### 验收标准

- 两个 IDE 插件功能对等，UI 一致性 ≥ 80%
- 5 名工程师试用 1 周，IDE 内写用例占比 ≥ 60%
- 插件市场上架（VSCode Marketplace + JetBrains Marketplace）

## 6. v1.x.E 商业版差异化

> 与开源版同源，但提供企业能力。社区贡献者不受影响，开源版始终是 self-hostable 的。

### 范围 In

- 进阶 SSO（SAML 2.0、OIDC + 自定义 IdP、SCIM 用户同步）
- 完整审计日志（每次 explore / heal / KB 写入 / 配置变更，含 actor / before / after）
- 数据合规导出（GDPR / 等保 2.0）
- 企业 SLA：4 小时响应 / 24 小时 patch
- 专属技术支持渠道
- 高级 Dashboard 报表（自定义看板、定时邮件）
- 多集群联邦（跨地域调度）
- 私有镜像仓 / 模型仓

### 不在商业版中

- 业务地图插件（v1.1 仍在开源版）
- 视频→图谱（v1.2 仍在开源版）
- 核心能力均在开源版

## 8. 依赖与风险

| 项 | 风险 | 缓解 |
|---|---|---|
| 业务地图与已有 Code-as-Action 互斥 | 用户不知道走哪条路径 | UI 明确提示；命中模板 / 未命中两条线 |
| 视频抽取依赖多模态精度 | 错误抽取会污染图谱 | 强制 review；置信度 < 阈值不入库 |
| IDE 插件维护两套代码 | 工程量大 | LSP 抽出共用核心；UI 各自实现 |
| 商业版 vs 开源版边界模糊 | 社区疑虑 | 公开 charter：核心永远开源；商业仅围绕"运营 / 合规 / 服务" |
